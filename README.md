# 🍡 鸣潮团子竞速赛 — WuWa Dango Racing 2026

> **《鸣潮》"团子大作战"活动 · 模拟竞速比赛程序**
>
> 使用 Rust 编写的《鸣潮》团子大作战竞速比赛模拟器。

---

## 📖 概述

本项目模拟《鸣潮》游戏活动 **"团子大作战"** 中的竞速比赛。可爱的团子们在环形赛道上竞速，每个团子都有独特的技能影响其移动方式。模拟器运行比赛，通过蒙特卡洛模拟计算胜率，并支持多阶段赛制：

- **小组赛** — 3 组每组 6 个团子，前 4 名晋级，后 2 名进入谢幕赛
- **淘汰赛** — 淘汰赛制（尚未实现）
- **总决赛** — 冠军争夺战（尚未实现）
- **谢幕赛** — 为被淘汰团子举办的表演赛（尚未实现）

---

## 🏁 功能特色

- **比赛模拟** — 完整的回合制比赛逻辑，包含掷骰子、堆叠和赛道机制
- **独特团子技能** — 每个团子都有独特的技能，在比赛中触发
- **赛道地图** — 32 格的环形赛道，包含加速/阻遏装置、时空裂隙等特殊格子
- **布大王机制** — 特殊团子"布大王"从终点向起点反向移动
- **团子堆叠** — 团子落在同一格时会堆叠，移动时携带上方所有团子
- **确定性模式** — 支持种子随机数生成器，确保可复现模拟
- **调试可视化** — debug 模式下显示每轮赛道状态
- **蒙特卡洛支持** — 架构设计支持批量模拟和概率统计

## 🎲 比赛规则

### 骰子

- 普通骰子：`[1, 1, 2, 2, 3, 3]`
- 布大王骰子：`[1, 2, 3, 4, 5, 6]`

### 赛道

环形赛道共 32 格，从 `(1,C)`（起点）到 `(32,C)`（终点）：

| 格子类型                    | 效果                     |
| --------------------------- | ------------------------ |
| **C** (Common)              | 无特殊效果               |
| **A** (Accelerate/推进装置) | 向前移动 1 格            |
| **D** (Decelerate/阻遏装置) | 向后移动 1 格            |
| **H** (Hole/时空裂隙)       | 随机重新堆叠此格上的团子 |

对布大王而言，推进装置使其后退（向终点靠近），阻遏装置使其前进（向起点靠近）。

### 堆叠

团子落在已有团子的格子上时，会堆叠在最上方。移动时，团子携带堆叠在其上方的所有团子一起移动。

### 布大王

- 第 3 回合从终点加入比赛
- 向起点方向**反向**移动
- 始终处于堆叠最底层
- 每轮结束时如果与最后一名分开，传送回终点
- 赛道机制对其生效，但加速/减速方向相反

## 🚀 快速开始

### 前提条件

- Rust 2024 edition
- Cargo

### 构建 & 运行

```bash
# 调试构建（含赛道可视化输出）
cargo run

# 发布构建（快速运行，无调试输出）
cargo run --release

# 指定随机种子（确定性模式）
cargo run -- <种子值>
# 例如
cargo run -- 42

# 运行测试
cargo test
```

### 当前运行行为

当前 `main()` 模拟 **小组赛 A 组** 比赛（达妮娅、西格莉卡、绯雪、卡提希娅、菲比、陆·赫斯），分为上下半场：

1. **上半场** — 所有团子从起点出发
2. **下半场** — 继承上半场结束时的位置，每个团子需再到达终点一次

## ⚙️ 技术要点

- **共享状态**: 使用 `Rc<RefCell<T>>` 实现比赛过程中团子状态的共享可变访问
- **Ambassador Crate**: 使用 `ambassador` crate 实现 `MyRng` 的委托模式
- **特性门控**: 调试可视化由 `#[cfg(debug_assertions)]` 保护，发布构建无性能开销
- **宏辅助**: `impl_run_for_dango_helper!` 宏自动生成 `Dango` 枚举到各变体的方法分发
